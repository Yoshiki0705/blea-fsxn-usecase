# FSx for ONTAP BLEA ユースケース選択ガイド

## デシジョンツリー

```
お客様の最初の課題は何ですか？
│
├── 「ファイルサーバーのデータを分析したい」
│   └── → Spec A: Data Analytics
│       条件: Glue + Athena で SQL 分析。データコピー不要。
│       コスト: ~$500/月 (dev), ~$1,500/月 (prod)
│       期待効果: データ活用率 0% → 分析可能に
│
├── 「ランサムウェアからデータを守りたい」
│   └── → Spec B: Cyber Resilience
│       条件: 金融・公共。管理者アカウント侵害を想定。
│       コスト: ~$700/月 (3アカウント合計)
│       期待効果: RPO 24h→1h, 管理者権限でも削除不可
│
├── 「支所からのファイルアクセスが遅い」
│   └── → Spec G: FlexCache Distributed
│       条件: 本庁↔支所の WAN 遅延問題。
│       コスト: ~$800/月 + cross-region転送
│       期待効果: ファイルオープン 200ms→<5ms
│
└── 「VMware/オンプレからAWSに移行したい」
    └── → Spec H: Modernization Platform
        条件: 共有ストレージが必要な複数コンピュート。
        コスト: ~$500/月 (最小), ~$2,000/月 (全パターン)
        期待効果: 単一ストレージで EC2/Lambda/ECS 統合
```

## 組み合わせパターン

| 顧客タイプ | 推奨組み合わせ | 理由 |
|-----------|-------------|------|
| 地方自治体 | B + G + A | セキュリティ + 分散拠点 + データ活用 |
| 金融機関 | B + A | ランサムウェア対策 + 規制対応分析 |
| 製造業 | H + A | レガシー移行 + IoT/ファイルデータ分析 |
| 教育機関 | G + A | キャンパス間ファイル共有 + 分析 |
